Symptome de votre AntiPattern

La technologie évolue si rapidement que les développeurs ont du mal à suivre les versions actuelles du logiciel et à trouver des combinaisons de versions de produit qui fonctionnent ensemble. Étant donné que chaque ligne de produits commerciaux évolue grâce aux nouveaux produits, la situation est devenue de plus en plus difficile pour les développeurs. Trouver des versions compatibles de produits qui interopèrent avec succès est encore plus difficile. Java est un exemple bien connu de ce phénomène, avec de nouvelles versions sortant tous les quelques mois. Par exemple, au moment où un livre sur Java 1.X passe à la presse, un nouveau kit de développement Java obsolete les informations. Java n'est pas seul; De nombreuses autres technologies participent également à Obsolescence continue.

Traitement

Un facteur important de stabilisation du marché des technologies est celui des normes de systèmes ouverts. Une norme de consortium est le produit d'un consensus de l'industrie qui exige du temps et des investissements. Les initiatives de commercialisation conjointe renforcent la sensibilisation et l'acceptation des utilisateurs au fur et à mesure que les technologies évoluent vers le grand public. Il ya une inertie inhérente dans ce processus qui profite aux consommateurs, car une fois qu'un produit du fournisseur est conforme à une norme, le fabricant est peu susceptible de changer les caractéristiques conformes du produit. Les avantages d'une technologie rapidement obsolète sont transitifs. Les architectes et les développeurs doivent dépendre des interfaces qui sont stables ou qu'ils contrôlent. Les normes de systèmes ouverts donnent une certaine stabilité à un marché de la technologie autrement chaotique.
